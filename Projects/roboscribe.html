RoboScribe

Language-guided policy generation for robosuite manipulation tasks. Describe a task in natural language → the system designs phased strategies with human approval → an agent writes code, rolls out in MuJoCo, diagnoses failures from trajectory metrics, and iterates until policies meet a success threshold.

Yuer Tang, Bree Chen

UCLA

Motivation

Writing robot policies shouldn’t take hours. RoboScribe turns natural language into executable robot control code — automatically generating, testing, and debugging policies in simulation so researchers can focus on ideas, not boilerplate.

Results

Success rates across four robosuite tasks of increasing geometric complexity. All runs use Qwen, the Panda robot with OSC_POSE controller, and no human feedback during code iteration.

Key findings

  • Success correlates with geometric precision required — translational tasks (Lift, Stack) succeed reliably, while tasks involving quaternions and coupled contact (NutAssembly, Door) fail.
  • The diagnostic system correctly identifies failures in every case — the bottleneck is code synthesis, not failure detection.
  • Iterative revision helps for medium tasks (Stack: 0%→100% in one revision) but does not improve hard tasks — each revision is a new sample from the same distribution, not a targeted fix.
  • Human feedback during code iteration does not improve outcomes. The effective intervention point is phase design, where human strategy expertise has the highest leverage.

Run it locally

Install

# Python backend (includes all dependencies)
pip install -e "roboscribe/[sim]"

# React frontend
cd roboscribe/src/roboscribe/frontend && npm install

Run

# Terminal 1 — Backend
cd roboscribe/src/roboscribe
uvicorn roboscribe.server.main:app --host 0.0.0.0 --port 8000

# Terminal 2 — Frontend
cd roboscribe/src/roboscribe/frontend
npm run build && npm run preview

Open http://localhost:4173. To preview this documentation site: cd docs && python3 -m http.server 8080 → http://localhost:8080.
